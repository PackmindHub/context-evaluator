# Plan: Add macOS Code Signing & Notarization to Release Workflow

## Context

Currently, the release workflow (`.github/workflows/ci.yml`) builds all 4 platform binaries on `ubuntu-latest` via Bun's cross-compilation and publishes them unsigned to GitHub Releases. macOS users get Gatekeeper warnings when running unsigned binaries. We need to add Apple code signing and notarization for the two macOS binaries (darwin-arm64, darwin-x64).

## Approach

Restructure the release workflow into 3 jobs (instead of current 1):

1. **build** - Build all binaries on ubuntu (existing logic)
2. **sign-macos** - Sign & notarize macOS binaries on `macos-latest`
3. **release** - Create GitHub Release with signed macOS + unsigned Linux/Windows binaries

## Changes

### 1. Create `entitlements.plist`

**New file:** `entitlements.plist` (project root)

Minimal entitlements for a CLI tool - just needs the hardened runtime allowance for unsigned executable memory (required by Bun-compiled binaries):

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>com.apple.security.cs.allow-jit</key>
    <true/>
    <key>com.apple.security.cs.allow-unsigned-executable-memory</key>
    <true/>
</dict>
</plist>
```

### 2. Modify `.github/workflows/ci.yml`

Split the `release` job into 3 jobs:

#### Job: `build` (runs-on: ubuntu-latest)
- Same as current release job: checkout, setup bun, install deps, `bun run build`
- **New**: Upload `dist/bin/*` as artifact named `binaries`

#### Job: `sign-macos` (runs-on: macos-latest, needs: build)
- Download the `binaries` artifact
- Install Apple certificate to temporary keychain
- Sign both `context-evaluator-darwin-arm64` and `context-evaluator-darwin-x64` with `codesign`
- Notarize both executables via `xcrun notarytool`
- Upload signed macOS binaries as artifact `macos-signed`

#### Job: `release` (runs-on: ubuntu-latest, needs: [build, sign-macos])
- Download `binaries` artifact (for linux-x64, windows-x64)
- Download `macos-signed` artifact (for darwin-arm64, darwin-x64)
- Create GitHub Release with all 4 binaries (macOS ones are signed)

### 3. Required GitHub Secrets

These must be configured in the repository settings (same as the reference project):

- `MACOS_CERTIFICATE` - Base64-encoded .p12 certificate
- `MACOS_CERTIFICATE_PASSWORD` - Password for the .p12 certificate
- `MACOS_SIGNING_IDENTITY` - Developer ID identity string
- `APPLE_ID` - Apple ID email for notarization
- `APPLE_ID_PASSWORD` - App-specific password for notarization
- `APPLE_TEAM_ID` - Apple Developer Team ID
