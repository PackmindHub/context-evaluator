# Plan: Tree View for Context Tab

## Context

The Context tab currently shows a flat list of context files grouped by agent (AGENTS.md, Claude Code, GitHub Copilot, Cursor). The user wants an alternative **tree view** that displays the same files as a navigable directory hierarchy, with a toggle to switch between the two views. Clicking a file in the tree should open the existing content browser modal with rendered markdown.

## Approach

### 1. Install Chakra UI v3

```bash
cd frontend && bun add @chakra-ui/react @emotion/react
```

Chakra UI v3 uses Ark UI under the hood for the TreeView component. We'll use `createTreeCollection` and `TreeView.*` components from the docs the user provided.

**Style conflict mitigation:**
- Wrap **only** the tree view component in a scoped `ChakraProvider` (not the entire app) to minimize global style interference
- Use `resetCSS={false}` to prevent Chakra from overriding Tailwind's global styles
- Apply custom theme overrides to match the existing dark slate palette

### 2. Create tree data builder utility

**New file: `frontend/src/lib/build-context-tree.ts`**

- Define `TreeNode` interface: `{ id: string; name: string; type?: string; path?: string; children?: TreeNode[] }`
- `buildContextTree(contextFiles, skills, linkedDocs)` function:
  1. Collect all items into flat `{ path, type }` entries
  2. Split each path on `/` to get segments
  3. Walk/create nested directory nodes, insert file leaf at final segment
  4. Sort: directories first (alphabetically), then files (alphabetically)
  5. Return `TreeNode[]` (root-level nodes)

### 3. Create `ContextTreeView` component

**New file: `frontend/src/components/ContextTreeView.tsx`**

- Uses Chakra UI's `TreeView.Root`, `TreeView.Tree`, `TreeView.Node` with `createTreeCollection`
- Scoped `ChakraProvider` wrapping only this component
- Folder/file icons using existing SVG icon patterns from `ContextTab.tsx`
- Custom theme to match slate/dark palette:
  - Background transparent (inherits glass-card parent)
  - Text colors: slate-300 for names, slate-500 for metadata
  - Hover: `bg-slate-800/30`
  - Selected: `bg-indigo-600/30` (matches `ContentBrowserModal` selection style)
- Props: `contextFiles`, `skills`, `linkedDocs`, `onFileClick(path, type)`
- Uses `TreeView.Node` shortcut pattern for recursive rendering
- File type badges on leaf nodes (agents, claude, rules, skills, etc.)

### 4. Modify `ContextTab.tsx` - Add view toggle

- Add `viewMode` state: `"list" | "tree"` (default: `"list"`)
- Add toggle button group at top-right of the component (before section cards):
  - List icon button (active when `viewMode === "list"`)
  - Tree icon button (active when `viewMode === "tree"`)
  - Styled: `bg-slate-800/50 rounded-lg` container, active state `bg-slate-700 text-slate-200`
- Conditionally render:
  - `viewMode === "list"` → existing SectionCard blocks (unchanged)
  - `viewMode === "tree"` → `<ContextTreeView>` component

### 5. Wire file click to existing modals

When `onFileClick(path, type)` is called from the tree:
- For context files (agents, claude, copilot, rules, cursor-rules): open `ContextFilesBrowserModal` with all files, pre-selecting the clicked file
- For skills: open `SkillsBrowserModal` pre-selecting the clicked skill
- For linked docs: open `LinkedDocsBrowserModal` pre-selecting the clicked doc

**Modify `ContentBrowserModal.tsx`:**
- Add optional `initialSelectedId?: string` prop
- In the `useEffect` that runs on `isOpen`, if `initialSelectedId` is provided, find the matching item index and set `selectedItemIndex` to it

**Modify `ContextFilesBrowserModal.tsx`:**
- Add optional `initialSelectedPath?: string` prop, pass through as `initialSelectedId` to `ContentBrowserModal`

**Modify `SkillsBrowserModal.tsx` and `LinkedDocsBrowserModal.tsx`:**
- Add optional `initialSelectedPath?: string` prop for pre-selection

### 6. Rebuild CSS

```bash
cd frontend && bun run css:build
```
