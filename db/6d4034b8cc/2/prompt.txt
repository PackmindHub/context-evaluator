# Plan: Add "Context" Tab with Grouped Sections Layout

## Context

The "Context Files" section (AGENTS.md, CLAUDE.md + Rules, Copilot Instructions, Skills, Linked Docs) currently lives inside the Summary tab as a flat 5-card grid. The user wants to:
1. Move it into its own "Context" tab next to "Summary"
2. Redesign the layout with grouped sections and headers for better readability

## UI Design: Grouped Sections

Two logical groups with section headers:

### Group 1: "Context Documentation" (or "Documentation Files")
- AGENTS.md files
- CLAUDE.md + Rules files
- Copilot Instructions files

Each row shows: file type icon + label, count badge, list of file paths, and a "Browse" button.

### Group 2: "Extensions"
- Skills (name + directory + description)
- Linked Docs (path + source + summary)

Each row shows: type icon + label, count badge, list of items, and a "Display all" button.

### Layout approach
- Each group is a `glass-card` section with a section header (icon + title + total count for group)
- Inside each group, items are displayed as clean rows/list items (not cards) with consistent spacing
- File paths shown inline, expandable if many
- Empty groups are hidden entirely
- If no context files at all, show an empty state message
