# Plan: Block evaluation of repos exceeding 1M lines of code (cloud mode)

## Context

The cloud-hosted version of the evaluator is vulnerable to abuse or accidental overload from very large repositories. Scanning a repo with millions of lines of code triggers expensive AI evaluator calls. This change adds a guard that rejects repos exceeding 1,000,000 LOC — only in cloud mode, so local/CLI usage is unaffected.

The `cloc` tool already runs during context identification and produces `clocOutput` (JSON with `SUM.code` = total LOC). We leverage this existing data.

### 3. Add LOC guard in the evaluation engine

**File**: `src/shared/evaluation/engine.ts` (after `identifyProjectContext()` + context logging, ~line 1005, before Step 3)

```typescript
// Enforce repository size limit in cloud mode
if (options.cloudMode && contextResult.clocOutput) {
    try {
        const clocData = JSON.parse(contextResult.clocOutput);
        const totalLOC: number = clocData.SUM?.code || 0;
        const maxLOC = Number(process.env.MAX_REPO_LOC) || 1_000_000;
        if (totalLOC > maxLOC) {
            const error = new Error(
                `Repository too large for cloud evaluation: ${totalLOC.toLocaleString()} lines of code exceeds the ${maxLOC.toLocaleString()} line limit.`
            );
            (error as any).code = "REPO_TOO_LARGE";
            throw error;
        }
    } catch (err) {
        if (err instanceof Error && (err as any).code === "REPO_TOO_LARGE") throw err;
        // Malformed cloc output → fail-safe, allow evaluation
    }
}
```

- **Fail-safe**: If cloc is unavailable or output is malformed, evaluation proceeds.
- **Configurable**: `MAX_REPO_LOC` env var overrides the 1M default.
- **Error propagation**: Job manager already extracts `error.code` at `job-manager.ts:434`.
