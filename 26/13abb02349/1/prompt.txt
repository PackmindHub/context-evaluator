# Plan: Fix Evaluate Impact UX - Show Started Message + Progress Link

## Context

When "Evaluate Impact" is clicked on a remediation history card, the frontend:
1. Calls `POST /api/remediation/:id/evaluate` to queue a new evaluation job
2. Opens an `EventSource` SSE connection to `/api/evaluate/:jobId/progress` to track completion
3. On SSE error → marks impact eval as "failed"

The problem: SSE fails with `ERR_INCOMPLETE_CHUNKED_ENCODING` (stream cut off during a long evaluation). This falsely shows "Impact evaluation failed" even though the backend job is still running.

The user wants: (1) a clear "evaluation started" message and (2) a link to the full evaluation report page (`/evaluation/:jobId`) to track progress independently of the SSE stream.

## Approach

Minimal fix in two files:

1. **`RemediationHistory.tsx`** — after the API call returns `result.jobId`, immediately store it in state (so the card can link to it). Fix the SSE error handler to **not** mark as "failed" — if we have a jobId, the eval is running in the background.

2. **`RemediationHistoryCard.tsx`** — add `impactJobId?: string` prop. In the "running" UI state, show "Evaluation started" with a `<a>` link to `/evaluation/{impactJobId}`. This replaces just the spinner/label.

## Critical Files

- `frontend/src/components/RemediationHistory.tsx` — orchestrates impact eval, manages SSE, sets state
- `frontend/src/components/RemediationHistoryCard.tsx` — renders the "Evaluate Impact" button and status row
